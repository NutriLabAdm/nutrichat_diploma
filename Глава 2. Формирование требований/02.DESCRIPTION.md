# Глава 2. Формирование требований и архитектуры платформы

## Задача 2. Сформировать функциональные и нефункциональные требования к платформе поведенческих интервенций

## Цель главы
Сформировать функциональные и нефункциональные требования к платформе, определить архитектурные границы, обеспечить интеграцию между сервисами экосистемы и перспективными модулями.

### 2.1. Цели и границы системы

**Предпосылки:** Экосистема Нутричат объединяет в своём контуре несколько систем на разных стадиях разработки. Для обеспечения масштабируемости и возможности подключения перспективных модулей (coach, community, genes, gut) необходимо спроектировать единую архитектуру с централизованной системой авторизации, общим хранилищем данных и оркестрацией взаимодействия через AMQP и MCP протоколы.

**Цели подраздела:** Определить границы системы, бизнес-цели и KPI для оценки эффективности платформы.

- Бизнес-цели платформы
- Границы системы и допущения
- Ключевые показатели эффективности (KPI)

### 2.3. Нефункциональные требования системы

**Предпосылки:** Платформа работает с персональными данными пользователей (здоровье, питание, цели), что требует соответствия GDPR и ФЗ-152. При этом чувствительные данные не планируется хранить на серверах — профиль, история питания, результаты анализов, данные поведенческих интервенций предполагается хранить на стороне пользователя, в том числе в защищённом виде с использованием технологии блокчейн. Высокая нагрузка от AI-сервисов и CV-распознавания требует асинхронной обработки. Микросервисная архитектура с AMQP обеспечит масштабируемость.

**Цели подраздела:** Определить требования к производительности, безопасности, масштабируемости и надёжности системы.

| Требование | Значение | Обоснование |
|------------|---------|------------|
| **Производительность** | UI < 2 сек, AI-ответы < 5 сек | UX |
| **Безопасность** | AES-256, GDPR-совместимость | Персональные данные |
| **Масштабируемость** | 10,000+ пользователей | Рост |
| **Доступность** | uptime 99.5%, mobile-first | Основная платформа — мобильные |
| **Надёжность** | Восстановление < 15 мин | SLA |

**Обоснование выбора VPS:** Указанные требования к нефункциональным характеристикам позволяют использовать VPS для размещения платформы. При начальной нагрузке до 10,000 пользователей конфигурация с 4 ядрами CPU, 8 GB RAM и 100 GB SSD обеспечит необходимую производительность и масштабируемость. При росте нагрузки возможен переход на выделенные серверы или Kubernetes-кластер.

**Примерные требования к VPS:**

| Параметр | Минимальные | Рекомендуемые |
|---------|------------|--------------|
| CPU | 2 ядра | 4 ядра |
| RAM | 4 GB | 8 GB |
| SSD | 50 GB | 100 GB |
| OS | Ubuntu 22.04 | Ubuntu 22.04 |
| DB | PostgreSQL 14 | PostgreSQL 16 + Redis |
| Backup | Ежедневно | Ежедневно + replica |

### Scrum vs AI-agents: трансформация ролей и артефактов

| Scrum (классика) | → | AI-agents (новая) | Изменение |
|-------------------|---|-----------------|----------|
| **Роли** | | | |
| Product Owner | → | AI Product Owner + Prompt Engineer | Слияние |
| Scrum Master | → | AI Orchestrator | Модификация |
| Developer | → | AI Code Agent + AI Test Agent | Слияние |
| QA Engineer | → | AI Test Agent | Исчезновение |
| DevOps | → | AI DevOps Agent | Модификация |
| Designer | → | AI Design Agent | Модификация |
| | | | |
| **Артефакты** | | | |
| Product Backlog | → | CODE_PROMTS.md | Слияние |
| Sprint Backlog | → | PROMPT_TEMPLATES.md | Модификация |
| Increment | → | Generated Code + Tests | Слияние |
| | | | |
| Документация | → | Auto-generated docs | Автоматизация |
| Тесты | → | AI-generated tests | Автоматизация |
| Code Review | → | AI Review Agent | Автоматизация |
| Deployment | → | AI Deployment Agent | Автоматизация |

**Ключевые изменения:**
- **Слияние:** Product Owner + Prompt Engineer → единая роль
- **Исчезновение:** Отдельный QA Engineer не нужен (AI Test Agent)
- **Автоматизация:** Documentation, Tests, Code Review, Deployment |

### AI_policy для ведения разработки

**Обоснование:** В рамках solo-разработки используется подход AI-first development с использованием AI-агентов:

1. **AI Orchestrator** — координирует работу агентов
2. **Agent code generation** — генерация кода по промптам
3. **Agent documentation** — автодокументирование
4. **Agent testing** — генерация тестов

**Стандарты:**
- CODE_PROMTS.md — общие промпты
- PROMPT_TEMPLATES.md — шаблоны
- CHANGELOG.md — версионирование

**CI/CD:** Автоматическое обновление артефактов при изменении промптов.
