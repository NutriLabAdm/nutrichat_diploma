# Глава 3. Проектирование архитектуры системы

## Задача 3. Спроектировать архитектуру системы с учётом концепции цифровых двойников пользователей

## Цель главы
Описать архитектурный подход к платформе Нутричат, спроектировать общую архитектуру и взаимодействие модулей, включая экспериментальный модуль с цифровыми двойниками.

### 3.1. Архитектурный подход

**Предпосылки:** Экосистема Нутричат включает разнородные сервисы с различными требованиями к нагрузке, задержке (latency) и надёжности. AI-сервисы (LLM, CV-распознавание) характеризуются асинхронной обработкой и высоким потреблением ресурсов, тогда как core-сервисы (трекинг, авторизация) требуют синхронного отклика с минимальной задержкой. Платформа работает с персональными данными (PII), что накладывает требования к шифрованию, compliance (GDPR/ФЗ-152) и аудиту доступа. Гибридная архитектура (микросервисы для AI-компонентов + синхронный API для критических путей) обеспечивает оптимальный баланс между масштабируемостью, отказоустойчивостью и безопасностью.

**Цели подраздела:** Определить архитектурный подход, обосновать выбор гибридной архитектуры, сформулировать принципы проектирования.

- Гибридная архитектура: микросервисы + монолит
- Принципы: модульность, слабая связанность, высокая связность
- Обоснование выбора подхода

### 3.3. Модель данных и потоки информации

**Предпосылки:** Система работает с персональными данными пользователей, историей питания, целями и результатами экспериментов. Необходимо определить структуру БД, потоки данных и интеграции с внешними системами. ER-модель должна поддерживать масштабирование и обеспечивать безопасность данных.

**Цели подраздела:** Определить ER-диаграмму основных сущностей, потоки данных, интеграции с внешними API.

Основные сущности платформы:
- **Users** — пользователи системы
- **Profiles / Digital Twins** — профили пользователей (статические данные) и цифровые двойники (динамическая модель поведения)
- **FoodEntries** — записи питания и трекинг
- **Goals** — цели пользователей
- **BiometricLogs** — биометрические данные (вес, давление, пульс)
- **BehavioralLogs** — поведенческие данные (активность, сон, стресс)
- **FinancialLogs** — финансовые данные (расходы на питание, добавки)

*Полная ER-диаграмма со всеми сущностями — в Приложении 3.2.*

**Цели подраздела:** Спроектировать модель Digital Twin, архитектуру обновле��ия, связь с AI-агентом.
