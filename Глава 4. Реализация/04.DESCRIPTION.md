### 4.1.3. Безопасность и защита данных

**Предпосылки:** Персональные данные пользователей (вес, питание, цели) требуют особой защиты в соответствии с законодательством.

**Цели подраздела:** Обеспечить конфиденциальность и безопасность данных.

- **Локальное хранение:** Чувствительные данные хранятся на устройстве или USB носителе пользователя в зашифрованном виде
- **Шифрование:** AES-256 для локальной базы
- **Минимум на сервере:** На сервере — только оркестрация сервисов и моделей и обезличенная статистика
- **GDPR/152-ФЗ:** Соответствие требованиям законодательства

### 4.1.4. Подход к разработке с использованием AI-агентов

**Предпосылки:** AI-агенты ускоряют разработку, особенно для рутинных задач и документации.

**Цели подраздела:** Определить роль AI-агентов в процессе разработки.

- **Генерация кода:** AI помогает писать boilerplate код, тесты, документацию
- **Анализ кода:** Поиск уязвимостей, оптимизация, рефакторинг
- **Документирование:** Автогенерация API docs, комментариев
- **Тестирование:** Генерация unit и интеграционных тестов
- **Research:** AI-поиск по кодовой базе, документации

## 4.7. Интеграция компонентов

**Предпосылки:** Экосистема Нутричат состоит из множества сервисов, которые должны слаженно работать вместе.

**Цели подраздела:** Описать механизмы интеграции между сервисами.


| # | Компонент | Технология | Ключевые функции |
|---|-----------|------------|------------------|
| 1 | Web (www) | React/Next.js, TypeScript | Основной интерфейс, PWA |
| 2 | Mobile | React Native | iOS/Android приложение |
| 3 | Telegram Bot | aiogram | Чат-интерфейс |
| 4 | Auth Service | FastAPI + JWT | Регистрация, вход, refresh tokens, провайдеры (Google, Apple) |
| 5 | API Gateway | Kong/Nginx | Маршрутизация, SSL, rate limiting, auth middleware |
| 6 | Message Queue | RabbitMQ (AMQP) | События: UserCreated, FoodLogged, GoalUpdated, ExperimentStarted |
| 7 | Контейнеризация | Docker | Упаковка сервисов, изоляция, воспроизводимость |
| 8 | Оркестрация | Kubernetes | Масштабирование, self-healing, rolling updates |
| 9 | База данных | PostgreSQL + Redis | Основное хранилище, кэширование сессий |
| 10 | Векторная БД | Chroma/PGVector | RAG для персонализации |
| 11 | CI/CD | GitHub Actions | Автоматическая сборка, тестирование, деплой |
| 12 | Мониторинг | Prometheus + Grafana | Метрики, алерты, дашборды |
| 13 | Логирование | ELK Stack (Elasticsearch, Logstash, Kibana) | Централизованные логи |
